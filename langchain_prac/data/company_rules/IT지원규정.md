# IT지원규정

## 제1장 총칙

### 제1조 (목적)
이 규정은 회사의 IT 장비 지급, 소프트웨어 관리, 기술 지원에 관한 사항을 정하는 것을 목적으로 한다. 본 규정은 2024년 4월 1일부터 시행한다.

### 제2조 (적용 범위)
본 규정은 회사의 전 임직원에게 적용된다. 외부 협력사 인원에게는 프로젝트 기간 동안 제한된 범위의 장비를 대여할 수 있으며, 프로젝트 종료 시 즉시 반납해야 한다.

## 제2장 장비 관리

### 제3조 (노트북 지급)
1. 신규 입사자에게는 입사일에 업무용 노트북을 지급한다.
2. 개발직에게는 MacBook Pro 16인치(M4 Pro, 메모리 36GB, SSD 512GB)를 기본 사양으로 지급한다.
3. 비개발직에게는 MacBook Air 15인치(M4, 메모리 24GB, SSD 256GB) 또는 동급 Windows 노트북을 지급한다.
4. 업무 특성상 고사양이 필요한 경우(영상 편집, 3D 렌더링, ML 학습 등), 팀장 승인 후 상위 사양으로 지급할 수 있다.
5. 노트북 교체 주기는 3년이며, 3년 이상 사용한 노트북은 신규 장비로 교체 신청이 가능하다.

### 제4조 (모니터 및 주변기기)
1. 전 직원에게 27인치 4K 모니터 1대를 기본 지급한다.
2. 개발직은 모니터 2대까지 신청 가능하다.
3. 키보드, 마우스는 회사 기본 지급품을 사용한다. 개인 선호 장비를 사용하고자 하는 경우 자비로 구매한다.
4. 모니터 암, 노트북 거치대, USB 허브 등 부속 장비는 IT지원팀에 신청하여 지급받는다.

### 제5조 (장비 관리 의무)
1. 지급받은 IT 장비는 회사의 자산이며, 업무 목적으로만 사용해야 한다.
2. 장비 분실 시 즉시 IT지원팀(내선 5500)과 정보보안팀(내선 5555)에 신고해야 한다.
3. 고의 또는 중과실로 인한 장비 파손 시, 수리비의 50%를 본인이 부담한다.
4. 퇴직 시 모든 IT 장비를 IT지원팀에 반납해야 하며, 반납 확인 후 퇴직 절차가 완료된다.
5. 장비 반납 시 개인 데이터 백업은 본인 책임이며, IT지원팀은 반납 즉시 초기화를 진행한다.

## 제3장 소프트웨어 관리

### 제6조 (소프트웨어 설치)
1. 업무용 노트북에는 회사가 승인한 소프트웨어만 설치할 수 있다.
2. 승인된 소프트웨어 목록은 사내 포털의 "IT 자산 관리" 페이지에서 확인 가능하다.
3. 목록에 없는 소프트웨어 설치가 필요한 경우, IT지원팀에 설치 요청을 신청한다.
4. 설치 요청은 보안 검토 후 처리되며, 일반적으로 3영업일 이내에 처리된다.
5. 불법 소프트웨어(크랙, 미인가 라이선스)의 설치는 엄격히 금지하며, 적발 시 징계 대상이 된다.

### 제7조 (라이선스 관리)
1. 유료 소프트웨어 라이선스는 IT지원팀이 중앙에서 관리한다.
2. 현재 회사에서 제공하는 주요 라이선스:
   - **개발 도구**: JetBrains All Products Pack (전 개발직), GitHub Enterprise
   - **디자인 도구**: Figma Business (디자인팀)
   - **협업 도구**: Slack Business+, Notion Team (전 직원), Zoom Business
   - **오피스**: Microsoft 365 Business (전 직원)
   - **클라우드**: AWS, GCP (프로젝트별 별도 계정)
3. 라이선스가 필요한 경우 사내 포털에서 신청하며, 팀장 승인 후 IT지원팀이 할당한다.
4. 사용하지 않는 라이선스는 반기별로 회수한다.

## 제4장 기술 지원

### 제8조 (지원 요청)
1. IT 관련 지원 요청은 사내 포털의 "IT 헬프데스크" 또는 Slack의 #it-support 채널을 통해 접수한다.
2. 긴급한 경우 IT지원팀 내선(5500)으로 직접 연락할 수 있다.
3. 지원 요청 처리 우선순위:
   - **긴급**: 시스템 장애로 업무 불가 → 1시간 이내 초기 대응
   - **높음**: 주요 기능 오작동 → 4시간 이내 대응
   - **보통**: 설정 변경, 소프트웨어 설치 요청 → 1영업일 이내
   - **낮음**: 일반 문의, 개선 요청 → 3영업일 이내
4. 지원 요청 처리 후 만족도 조사를 실시하며, 결과는 IT지원팀 서비스 개선에 활용한다.

### 제9조 (VPN 및 원격 접속)
1. 재택근무 및 외부 근무 시 사내 시스템 접근은 반드시 회사 VPN을 통해야 한다.
2. VPN 계정은 입사 시 자동 발급되며, 설치 가이드는 사내 포털에서 확인 가능하다.
3. VPN 접속 시 2단계 인증(MFA)을 필수로 적용한다. 인증 수단은 회사 지급 OTP 앱을 사용한다.
4. VPN 접속에 문제가 있는 경우 IT지원팀에 문의한다. 계정 잠김 시 본인 확인 후 해제한다.
5. 해외에서의 VPN 접속은 사전에 IT지원팀에 허용 요청을 해야 한다. 미등록 국가에서의 접속은 자동 차단된다.

## 제5장 개발 인프라

### 제10조 (개발 환경)
1. 개발 서버는 프로젝트별로 할당하며, DevOps팀이 관리한다.
2. 개발/스테이징/프로덕션 환경을 분리 운영하며, 각 환경 간 데이터 이동은 DevOps팀의 승인이 필요하다.
3. 프로덕션 환경에 대한 직접 접근은 금지한다. 배포는 반드시 CI/CD 파이프라인을 통해야 한다.
4. CI/CD 파이프라인은 GitHub Actions를 기본으로 사용하며, 빌드·테스트·배포 자동화를 포함한다.
5. 프로덕션 배포는 최소 1명의 코드 리뷰어 승인 후 가능하다.

### 제11조 (클라우드 리소스)
1. 클라우드 리소스(서버, 데이터베이스, 스토리지 등)의 생성은 DevOps팀을 통해 요청한다.
2. 개발/테스트 목적의 리소스는 팀장 승인으로 생성 가능하며, 프로덕션 리소스는 부서장 승인이 필요하다.
3. 미사용 리소스는 월별로 점검하여 삭제하며, 비용은 프로젝트별로 정산한다.
4. 클라우드 비용이 프로젝트 예산의 80%에 도달하면 자동 알림이 발송되며, 100% 초과 시 추가 예산 승인이 필요하다.
5. 개발 환경의 GPU 서버(ML/AI 학습용)는 사전 예약제로 운영하며, 최대 연속 사용 시간은 72시간으로 제한한다.

### 제12조 (데이터베이스 관리)
1. 프로덕션 데이터베이스에 대한 직접 쿼리 실행은 금지한다. 읽기 전용 레플리카를 사용해야 한다.
2. 스키마 변경은 마이그레이션 스크립트를 작성하고, DBA 검토 후 적용한다.
3. 프로덕션 데이터를 개발/테스트 환경에 복제할 경우, 개인정보는 반드시 마스킹 처리한다.
4. 데이터베이스 백업은 매일 자동으로 실행되며, 백업 데이터는 30일간 보관한다.
5. 백업 복구 테스트는 분기별로 실시하며, 결과를 DevOps팀이 보고한다.
