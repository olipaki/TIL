# 지식재산권규정

## 제1장 총칙

### 제1조 (목적)
이 규정은 회사의 지식재산권(특허, 저작권, 영업비밀 등)의 창출, 보호, 활용에 관한 사항을 정하는 것을 목적으로 한다.

### 제2조 (적용 범위)
본 규정은 전 임직원에게 적용되며, 외부 협력사 인원에게는 별도의 지식재산권 관련 계약 조항이 적용된다.

## 제2장 직무 발명

### 제3조 (직무 발명의 귀속)
1. 임직원이 업무 수행 과정에서 발명한 것(직무 발명)은 회사에 귀속된다.
2. 직무 발명의 판단 기준: 업무 시간 중 개발, 회사 자원(장비, 데이터, 인프라) 사용, 업무 관련 기술 분야
3. 업무 외 시간에 개인 자원으로 개발한 것은 개인에게 귀속된다. 다만, 회사의 사업 영역과 직접적으로 경쟁하는 경우 사전 신고가 필요하다.
4. 입사 전 보유하고 있던 지식재산권은 개인에게 귀속되며, 입사 시 목록을 인사팀에 제출한다.

### 제4조 (발명 보상)
1. 직무 발명에 대해 회사는 발명자에게 보상금을 지급한다.
2. 출원 보상: 특허 출원 시 1건당 50만원
3. 등록 보상: 특허 등록 시 1건당 100만원
4. 실시 보상: 해당 특허가 매출에 기여한 경우, 연간 기여 매출의 1% (최대 1,000만원)
5. 발명자가 2인 이상인 경우 기여도에 따라 보상금을 분배한다.

## 제3장 소프트웨어 저작권

### 제5조 (소스 코드 관리)
1. 업무상 작성한 소스 코드의 저작권은 회사에 귀속된다.
2. 모든 소스 코드는 회사의 Git 저장소(GitHub Enterprise)에 관리한다.
3. 개인 저장소(개인 GitHub 등)에 회사 코드를 복사하거나 업로드하는 것을 금지한다.
4. 퇴직 시 업무 관련 소스 코드를 개인적으로 보유할 수 없다.

### 제6조 (오픈소스 사용)
1. 오픈소스 소프트웨어를 업무에 사용하는 경우, 라이선스 조건을 확인해야 한다.
2. GPL, AGPL 등 Copyleft 라이선스가 적용된 소프트웨어는 사용 전 법무팀의 검토를 받아야 한다.
3. MIT, Apache 2.0, BSD 등 Permissive 라이선스는 자유롭게 사용 가능하나, NOTICE 파일에 명시해야 한다.
4. 사용 중인 오픈소스 목록은 프로젝트별로 관리하며, 분기별로 법무팀에 보고한다.

### 제7조 (오픈소스 기여)
1. 회사 업무와 관련된 오픈소스 프로젝트에 기여(코드 기여, 이슈 등록 등)하는 것은 장려한다.
2. 기여 전 팀장의 사전 승인을 받아야 하며, 회사 기밀 정보가 포함되지 않도록 주의한다.
3. 새로운 오픈소스 프로젝트를 회사 명의로 공개하는 경우, CTO의 승인이 필요하다.
4. 오픈소스 기여 활동은 성과 평가의 기술 역량 항목에서 가점 요소로 반영한다.

## 제4장 영업 비밀

### 제8조 (영업 비밀 관리)
1. 영업 비밀은 고객 목록, 가격 정책, 사업 전략, 기술 로드맵, 미공개 재무 정보 등을 포함한다.
2. 영업 비밀에 해당하는 문서는 "기밀" 등급으로 분류하고, 정보보안규정에 따라 관리한다.
3. 영업 비밀의 외부 공개가 필요한 경우, 반드시 NDA(비밀유지계약)를 체결한 후 공개한다.
4. 영업 비밀 유출 시 민·형사상 법적 책임을 질 수 있으며, 징계규정에 따라 중징계 대상이 된다.
