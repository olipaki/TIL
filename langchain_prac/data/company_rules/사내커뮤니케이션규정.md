# 사내 커뮤니케이션 규정

## 제1장 총칙

### 제1조 (목적)
이 규정은 회사의 사내 커뮤니케이션 채널, 도구, 에티켓에 관한 사항을 정하여 효율적이고 건전한 소통 문화를 조성하는 것을 목적으로 한다.

### 제2조 (적용 범위)
본 규정은 전 임직원에게 적용되며, 외부 협력사 인원에게는 해당 채널의 이용 범위에 한하여 적용된다.

## 제2장 커뮤니케이션 채널

### 제3조 (공식 채널)
1. **Slack**: 일상 업무 소통의 기본 채널. 팀별, 프로젝트별, 주제별 채널을 운영한다.
2. **이메일**: 공식 문서, 대외 커뮤니케이션, 기록이 필요한 의사결정에 사용한다.
3. **Notion**: 문서 작성, 프로젝트 관리, 사내 위키 용도로 사용한다.
4. **Zoom/Google Meet**: 화상회의에 사용하며, 3인 이상의 회의는 녹화를 권장한다.
5. **사내 포털(portal.techstar.co.kr)**: 공지사항, 규정, 경비/휴가 신청, IT 지원 요청 등 행정 업무에 사용한다.

### 제4조 (Slack 운영 규칙)
1. 채널 네이밍 규칙:
   - 팀 채널: #team-{팀명} (예: #team-backend, #team-design)
   - 프로젝트 채널: #proj-{프로젝트명} (예: #proj-newapp)
   - 주제별 채널: #topic-{주제} (예: #topic-ai, #topic-devops)
   - 지원 채널: #it-support, #hr-support, #general
2. 업무 요청이나 의사결정은 스레드(thread)를 사용하여 맥락을 유지한다.
3. @channel, @here 멘션은 긴급하거나 전체에게 알려야 하는 경우에만 사용한다.
4. 업무 시간 외(22:00~08:00) 메시지 발송 시 예약 발송 기능을 사용하거나, 즉시 응답을 기대하지 않는다.
5. 개인 DM으로 업무 요청을 하는 것은 지양한다. 채널에서 소통하여 정보를 공유한다.

### 제5조 (이메일 규칙)
1. 이메일 제목에는 [카테고리]를 표시한다. 예: [요청], [공유], [보고], [긴급]
2. 수신자, 참조(CC), 숨은참조(BCC)를 적절히 구분하여 사용한다.
3. 전사 발송(all@techstar.co.kr)은 경영지원팀 또는 인사팀장 승인 후에만 가능하다.
4. 대외 이메일 발송 시 회사 서명 템플릿을 사용한다.
5. 기밀 등급의 내용은 이메일로 전송할 수 없다. 사내 DRM 시스템을 통해 공유한다.

## 제3장 회의 문화

### 제6조 (회의 원칙)
1. 회의에는 반드시 안건(agenda)을 사전에 공유한다.
2. 회의 시간은 기본 30분으로 설정하며, 필요 시 연장한다. 1시간을 초과하지 않도록 권장한다.
3. 회의록은 참석자가 돌아가며 작성하고, 회의 후 24시간 이내에 Notion에 공유한다.
4. 의사결정이 필요한 회의에서 결론이 나지 않은 경우, 후속 회의 일정과 담당자를 명확히 한다.
5. "회의 없는 시간대"를 설정한다: 매주 수요일 오전(10:00~12:00)은 전사 회의 금지 시간이다.

### 제7조 (화상회의 에티켓)
1. 카메라를 켜는 것을 권장하되, 강제하지 않는다.
2. 발언하지 않을 때는 마이크를 음소거한다.
3. 화면 공유 시 개인 정보(메신저 알림 등)가 노출되지 않도록 주의한다.
4. 화상회의 녹화 시 참석자 전원의 동의를 받아야 한다.

## 제4장 공지 및 정보 공유

### 제8조 (공지사항)
1. 전사 공지는 사내 포털의 공지사항 게시판과 Slack #general 채널에 동시 게재한다.
2. 부서별 공지는 해당 부서 Slack 채널에 게재한다.
3. 중요 공지(인사 발령, 정책 변경, 시스템 점검 등)는 이메일로도 발송한다.

### 제9조 (사내 위키)
1. Notion 사내 위키에 팀별 업무 매뉴얼, FAQ, 온보딩 가이드를 작성하고 유지한다.
2. 위키 문서의 최종 수정일이 6개월을 초과한 경우, 담당자에게 업데이트 알림을 발송한다.
3. 위키의 접근 권한은 문서별로 설정하며, 기밀 정보는 위키에 게재할 수 없다.
