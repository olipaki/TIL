# 정보보안규정

## 제1장 총칙

### 제1조 (목적)
이 규정은 회사의 정보 자산을 보호하고, 보안 사고를 예방하며, 관련 법령을 준수하기 위한 기준을 정하는 것을 목적으로 한다. 본 규정은 2024년 3월 1일부터 시행한다.

### 제2조 (적용 범위)
본 규정은 임직원, 계약직, 인턴, 파견직, 외부 협력사 인원 등 회사의 정보 시스템에 접근하는 모든 인원에게 적용된다. 외부 협력사 인원은 별도의 보안 서약서를 제출해야 하며, 보안 규정 위반 시 즉시 출입이 제한된다.

## 제2장 계정 및 접근 관리

### 제3조 (계정 관리)
1. 모든 시스템 계정은 개인별로 발급하며, 계정 공유는 엄격히 금지한다.
2. 비밀번호는 영문 대소문자, 숫자, 특수문자를 포함하여 최소 12자 이상으로 설정해야 한다.
3. 비밀번호는 90일마다 변경해야 하며, 최근 5회 사용한 비밀번호는 재사용할 수 없다.
4. 5회 연속 로그인 실패 시 계정이 자동으로 잠기며, 정보보안팀에 요청하여 해제해야 한다.
5. 퇴직자의 계정은 퇴직일 당일에 비활성화하고, 30일 후 완전히 삭제한다.

### 제4조 (접근 권한)
1. 시스템 접근 권한은 업무 수행에 필요한 최소 범위로 부여한다 (최소 권한의 원칙).
2. 접근 권한의 신청은 사내 포털에서 팀장 승인 후 정보보안팀이 처리한다.
3. 부서 이동 시 기존 부서의 접근 권한은 이동일로부터 7일 이내에 회수한다.
4. 관리자(Admin) 권한은 정보보안팀장의 승인이 필요하며, 분기별로 재검토한다.
5. 외부에서의 시스템 접근은 반드시 VPN을 통해야 하며, 2단계 인증(MFA)을 필수로 적용한다.

## 제3장 데이터 보안

### 제5조 (문서 보안 등급)
1. 사내 문서는 다음 3단계로 분류한다:
   - **일반**: 공개 가능한 정보 (예: 채용 공고, 회사 소개서)
   - **대외비**: 사내에서만 열람 가능한 정보 (예: 내부 회의록, 프로젝트 계획서)
   - **기밀**: 권한이 부여된 인원만 열람 가능 (예: 재무제표 초안, M&A 관련 자료, 고객 개인정보)
2. 대외비 이상의 문서는 외부 클라우드 서비스(Google Drive, Dropbox 등)에 업로드할 수 없다.
3. 기밀 문서는 반드시 사내 DRM(Digital Rights Management) 시스템을 적용하여 관리한다.
4. 문서 출력 시 보안 등급에 따라 워터마크가 자동으로 삽입된다.

### 제6조 (개인정보 보호)
1. 고객 및 직원의 개인정보는 개인정보보호법에 따라 수집 목적 범위 내에서만 사용한다.
2. 개인정보가 포함된 파일은 AES-256 이상의 알고리즘으로 암호화하여 저장해야 한다.
3. 개인정보의 외부 전송이 필요한 경우, DPO(Data Protection Officer)의 사전 승인을 받아야 한다.
4. 개인정보 처리 대장을 작성하고, 분기별로 갱신한다.
5. 개인정보 유출 사고 발생 시 즉시 정보보안팀에 신고하고, 24시간 이내에 관련 기관에 보고해야 한다.

## 제4장 장비 및 네트워크 보안

### 제7조 (장비 보안)
1. 업무용 노트북에는 반드시 화면 잠금을 설정해야 하며, 자동 잠금 시간은 5분 이내로 설정한다.
2. 개인 USB, 외장 하드 등 외부 저장 매체의 업무용 PC 연결은 금지한다.
3. 업무 데이터를 개인 기기(스마트폰, 태블릿, 개인 노트북)에 저장하는 것을 금지한다.
4. 노트북 분실 시 즉시 정보보안팀(내선 5555)에 신고해야 하며, 원격으로 데이터를 삭제할 수 있다.
5. 업무용 노트북에는 회사가 승인한 소프트웨어만 설치할 수 있으며, 승인 목록은 사내 포털에서 확인 가능하다.

### 제8조 (네트워크 보안)
1. 사내 Wi-Fi는 WPA3-Enterprise 방식으로 운영하며, 개인 인증서를 통해 접속한다.
2. 게스트 Wi-Fi는 방문객 전용이며, 사내 시스템에는 접근할 수 없다.
3. 사내 네트워크에서 P2P 프로그램, 토렌트 등의 사용은 금지한다.
4. 의심스러운 네트워크 트래픽이 감지되면, 정보보안팀이 해당 기기의 네트워크 접근을 즉시 차단할 수 있다.
5. 사내 서버 접근은 점프 서버(Bastion Host)를 통해서만 가능하며, 모든 접근 로그는 1년간 보관한다.

## 제5장 보안 교육 및 사고 대응

### 제9조 (보안 교육)
1. 전 임직원 대상 정보보안 교육을 연 2회(3월, 9월) 실시한다.
2. 개인정보 보호 교육은 연 1회 별도로 실시한다.
3. 신규 입사자는 입사 후 1주일 이내에 보안 교육을 이수해야 한다.
4. 보안 교육 미이수자는 사내 시스템 접근이 제한될 수 있다.

### 제10조 (보안 사고 대응)
1. 보안 사고 발생 시 즉시 정보보안팀(내선 5555, security@techstar.co.kr)에 신고한다.
2. 보안 사고 등급은 다음과 같이 분류한다:
   - **1등급(긴급)**: 개인정보 대량 유출, 랜섬웨어 감염, 시스템 해킹 — 발생 즉시 CISO에게 보고
   - **2등급(중요)**: 내부 정보 외부 유출, 계정 도용 — 1시간 이내 보고
   - **3등급(경미)**: 비밀번호 유출 의심, 의심 메일 수신 — 24시간 이내 보고
3. 보안 사고 대응 후 30일 이내에 재발 방지 대책을 수립하여 보고한다.

### 제11조 (제재)
1. 보안 규정 위반 시 위반 수준에 따라 다음과 같이 조치한다:
   - **경고**: 경미한 위반 (예: 화면 잠금 미설정)
   - **시정 조치 + 추가 보안 교육**: 반복적 경미 위반 또는 1회 중대 위반
   - **징계 위원회 회부**: 고의적 정보 유출, 개인정보 무단 사용 등 중대 위반
2. 외부 협력사 인원의 보안 규정 위반 시, 해당 협력사에 통보하고 출입을 제한한다.
